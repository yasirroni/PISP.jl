# ISP2026 tutorial

This tutorial covers the release-specific setup for building datasets from the 2026 Integrated System Plan (ISP) datafiles.

## Source files
`ParseISP.download_isp26_source_files(downloadpath; kwargs...)` downloads the source artefacts used by the ISP2026 parser:

- 2026 ISP Inputs and Assumptions workbook
- 2026 ISP generation and storage outlook
- 2026 ISP model
- 2026 ISP solar trace archive
- 2026 ISP wind trace archive
- AEMO 2025 IASR EV workbook

The EV workbook is required because the 2026 ISP `Battery & Plug-in EVs` sheet refers to the AEMO 2025 IASR EV workbook for detailed EV assumptions. The downloader saves it in `downloadpath` as `aemo-2025-iasr-ev-workbook.xlsx`, and `build_datasets(ParseISP.ISP2026(); ...)` validates that it is present before parsing.

If AEMO blocks programmatic download access, download the EV workbook in a browser from the official AEMO link and place it at the same path:

```text
downloadpath/aemo-2025-iasr-ev-workbook.xlsx
```

## Build example
The standard build path downloads the required source files, extracts the supporting archives, prepares auxiliary outlook files, prepares trace inputs, and writes the output datasets.

```julia
using ParseISP

download_root = joinpath(@__DIR__, "..", "data", "parseisp-downloads")
output_root = joinpath(@__DIR__, "..", "data", "parseisp-datasets")

ParseISP.build_datasets(
    ParseISP.ISP2026(),
    downloadpath = download_root,
    years        = [2026],
    output_root  = output_root,
    write_csv    = true,
    write_arrow  = false,
)
```

For repeated local runs, use `download_from_AEMO = false` when the downloaded source files are already present. Use `prepare_supporting_assets = false`, `prepare_outlook = false`, or `build_traces = false` only when those prepared files already exist from an earlier run.

## Financial-year schedules
For ISP2026, `years = [Y]` means the financial year from `Y-07-01` through `Y+1-06-30`.

Examples:

| `years` value | Schedule covered |
| --- | --- |
| `[2026]` | FY2026-27, from 2026-07-01 through 2027-06-30 |
| `[2030]` | FY2030-31, from 2030-07-01 through 2031-06-30 |
| `[2050]` | FY2050-51, from 2050-07-01 through 2051-06-30 |

Supported values are `2026:2050`. `years = [2025]` is rejected because FY2025-26 starts before the 2026 ISP trace sources. `years = [2051]` is rejected because the required trace sources end on 2051-06-30.

For `years = [2050]`, trace-based schedules use the available FY2050-51 traces. The 2026 VPP outlook workbooks provide columns through `2049-50`, so the ISP2026 parser carries the `2049-50` VPP outlook value forward for FY2050-51.

## Date-range schedules
`drange` can be used instead of `years` for explicit date windows. Each tuple is `(start, end)`, where each value may be a `Date`, `DateTime`, or string in `"DD-MM-YYYY"` format.

```julia
using ParseISP

ParseISP.build_datasets(
    ParseISP.ISP2026(),
    downloadpath = joinpath(@__DIR__, "..", "data", "parseisp-downloads"),
    drange       = [("01-07-2026", "30-09-2026")],
    output_root  = joinpath(@__DIR__, "..", "data", "parseisp-datasets"),
    write_csv    = true,
    write_arrow  = false,
)
```

`years` and `drange` are mutually exclusive, and one of them must be supplied.
