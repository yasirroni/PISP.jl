ISP 2024

PISP implements a data-construction workflow for the 2024 Integrated System Plan. PISP.build_ISP24_datasets reads the supported inputs, applies the package's parsers and mappings, and writes static asset tables and time-varying schedule tables.

New users should begin with the Quickstart to install PISP.jl, build one short date range, and check representative outputs. Start with the ISP 2024 data sources to understand the source families and local input expectations. The output tables page defines the generated dataset and its joins. The parameters and mappings page records package-defined conventions that affect interpretation.

Reading the generated data

The static tables describe assets and their comparatively stable attributes. The schedule tables describe selected quantities that vary by scenario and time. Interpret a schedule alongside the corresponding static table and its scenario and date fields rather than as a standalone asset inventory.

The working with PISP-generated outputs tutorial demonstrates this relationship with a local 2024 build. The validation and analysis sections of the ISP 2024 documentation provide release-specific checks and exploratory context for those sources and outputs.

Scope

The 2024 workflow is a data-preparation step for downstream power-system studies. It does not itself run dispatch, unit commitment, capacity expansion, or power-flow models. See assumptions and scope before treating generated data as study-ready.
